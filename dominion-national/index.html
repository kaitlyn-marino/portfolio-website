Project Details

Goal: Create a more user friendly website, implement responsive design and increase functionality.

Role: I designed mock-ups of the new webpages and layouts, created working prototypes of the mobile friendly version to use as the base for coding, coded the HTML, stylized the CSS, and ensured fluid responsive design. Please note, the search feature is not currently functional as the client did not want JavaScript incorporated yet.

Features: Added responsive design, utilized the full-width of the web page, added a mobile friendly menu navigation, re-designed certain features to be more user friendly, used PHP for the headers on each page, and created new layouts that help the user move through the website easier.

Challenges: The current version of the website was not responsive and many elements were not functioning properly. By creating a more user friendly layout, implementing responsive design and using clean code, I was able to transition the design to a responsive layout and ensure all elements were correctly functioning. I also was able to implement PHP for the headers on each page so they would all display exactly the same and the client will only have to update one file to change all pages.

Project Highlights

Responsive Design: Transitioned the design to be responsive and functional across all platforms. The original site lost functionality and re-positioned content out of order when viewed on mobile or tablet platforms.

Menu Navigation: Added a hamburger-style menu navigation to the mobile and tablet versions. When viewing the original site on these platforms, there was no way to navigate to the menu item pages.

Dropdown Items: There were multiple sets of dropdown items under the original navigation making it difficult for the user to navigate to each page. The design was updated to decrease the number of dropdown items and visually allow the user to click through the pages.

Footer Links: Updated the footer links to be more user friendly and include responsive design. The new footer allows for easier navigation and less space as the items in the footer are of less importance to the page.

Buttons & Click-Throughs: Added buttons for user click-through items. By adding buttons the user functions available on the page become more clear and allow the user to continue moving through the website easier and quicker. Buttons were stylized with hover commands on the desktop version so the user can quickly identify when they are positioned over the button. Telephone references were added to phone numbers for call links to populate.
